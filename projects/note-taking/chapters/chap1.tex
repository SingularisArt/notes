\subsection{Source Lessons}
\label{sub_sec:source-lessons}

This file is where I source all of my lessons. Here's a quick example:

\begin{minted}{tex}
  % Unit 1 Started
  \input{unit-1/unit-info.tex}
  \input{unit-1/lesson-1.tex}
  \input{unit-1/lesson-2.tex}
  \input{unit-1/lesson-3.tex}
  % Unit 1 Ended


  % Unit 2 Started
  \input{unit-2/unit-info.tex}
  \input{unit-2/lesson-1.tex}
  \input{unit-2/lesson-2.tex}
  \input{unit-2/lesson-3.tex}
  \input{unit-2/lesson-4.tex}
  \input{unit-2/lesson-5.tex}
  \input{unit-2/lesson-6.tex}
  % Unit 2 Ended


  % Unit 3 Started
  \input{unit-3/unit-info.tex}
  \input{unit-3/lesson-1.tex}
  \input{unit-3/lesson-2.tex}
  \input{unit-3/lesson-3.tex}
  \input{unit-3/lesson-4.tex}
  \input{unit-3/lesson-5.tex}
  \input{unit-3/lesson-6.tex}
  % Unit 3 Ended


  % Unit 4 Started
  \input{unit-4/unit-info.tex}
  \input{unit-4/lesson-1.tex}
  \input{unit-4/lesson-2.tex}
  \input{unit-4/lesson-3.tex}
  \input{unit-4/lesson-4.tex}
  \input{unit-4/lesson-5.tex}
  \input{unit-4/lesson-6.tex}
  % Unit 4 Ended
\end{minted}

Then, I call this file within my \textbf{master.tex} file instead of manually
adding each lesson to my \textbf{master.tex} file. Also, the majority of my
scripts use this file to add and remove lessons.

\label{sub_sec:current_course}

This is probebly the most important item. This is a symlink to the current
course that I'm currently working on. If you don't know, a symlink is just a

\subsubsection{Rofi Commands}
\label{sub_sub_sec:rofi_commands}

You can view this script
\hyperlink{https://github.com/SingularisArt/local/blob/master/scripts/school/rofi-commands.py}{here}.

This script is probebly my favorite. So, when I take notes on a new lesson, I
create a new file in a folder for the unit. EX: \textbf{unit-1/lesson-1.tex}.
Then, I add
that to a file called \textbf{source-lessons.tex}. Then, I source that in my
\textbf{master.tex} file. Here's where the interesting thing happens. This
script pops up a menu asking
what I want to do.

\begin{figure}[ht]
    \centering
    \includegraphics[width=\textwidth,height=\textheight,keepaspectratio]{images/rofi-commands-1.png}
    \caption{Rofi Commands Prompt}
    \label{fig:executive-branch-structure}
\end{figure}

When I select the \textbf{Current Lesson} option, it will add only the current
lesson that I'm working on to the \textbf{source-lessons.tex} file. Then, it
will compile my document for me.

When I select the \textbf{Last two lessons} option, it will add the last two
lessons to my \textbf{source-lessons.tex} file. Then, it will compile my
document for me. I found that if you get the last two lessons, it will compile
quickly and you will get a quick memory refresh from the last lesson.

When I select the \textbf{All lessons in Unit} option, it will add all of the
lessons in the current unit to my \textbf{source-lessons.tex} file. Then, it
will compile my document for me.

When I select the \textbf{All lessons} option, it will add all of the lessons to
the \textbf{source-lessons.tex} file. Then, it will compile my document for me.

After I select one of the options, it will give me this prompt.

\begin{figure}[h]
    \centering
    \includegraphics[width=\textwidth,height=\textheight,keepaspectratio]{images/rofi-commands-2.png}
    \caption{Rofi Commands Prompt}
    \label{fig:executive-branch-structure}
\end{figure}

If I select \textbf{Open PDF}, it will open the compiled PDF, otherwise, it will
just exit.

\subsubsection{Rofi Projects}
\label{sub_sub_sec:rofi_projects}

You can view this script
\hyperlink{https://github.com/SingularisArt/local/blob/master/scripts/school/rofi-projects.py}{here}.

This is a work in progress script, but my goal is to have it list all of my
current projects. Then, when I click on the project, it will give me these
options:

\begin{enumerate}
  \item Open Project in File Browser
  \item Open Project in Text Editor
  \item Open Project master.tex file in Text Editor
  \item Compile Project
  \item View Project master.pdf
  \item Finished Project
\end{enumerate}

When I select the first option, it will just, simply, open the project in my
file browser. When I select the second option, it will open the project in my
text editor. When I select the third option, it will open the project's
\textbf{master.tex} file in my text editor. When I select the fourth option, it
will compile the project. When I select the fifth option, it will open the
\textbf{master.pdf} file in my pdf viewer. When I select the sixth option, it
will mark the project as finished.
